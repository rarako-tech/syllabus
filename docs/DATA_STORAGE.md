# データの保存方針

Syllabus プロジェクトで「何を」「どこに」保存するかの合意メモ。試作段階（2026）時点。

## 最優先で守るもの

**運用データ** — 教員が入力・更新するシラバス関連情報。

- シラバス基本情報
- 各回（目標・日付・準備物など）
- タイムスケジュール・教案メモ
- スライド・参照リンク

紹介用の図解ページやデモデータより、こちらを正とする。

---

## 保存先の一覧

| 種類 | 保存先 | リポジトリ | Neon（DB） | 外部サービス |
|------|--------|:----------:|:----------:|:------------:|
| シラバス・各回・教案・リンク | 運用データ | — | ○ | — |
| スライド・教材（URL） | 運用データ | — | URL のみ | Canva / Drive 等が正本 |
| ユーザー・組織 | 認証・テナント | — | ミラー | **Clerk が正本** |
| DB スキーマ | インフラ | ○ | 適用先 | — |
| デモシラバス | 開発用 | ○（コード） | — | — |
| 図解 HTML・キャプチャ | 公開・説明 | ○（正本） | — | Surge / `/overview` で公開 |
| 秘密鍵・接続文字列 | 設定 | —（`.gitignore`） | — | Vercel 環境変数 |
| Claude の質問・回答 | — | 保存しない | 保存しない | クリップボード経由 |
| PDF/PPT アップロード | 将来 | — | URL のみ（予定） | Blob 等（未実装） |

---

## Neon（DB）に入れるもの

**4ペイン分のテキスト・URL・メタデータのみ。**

- ① 各回で教える内容
- ② タイムスケジュール・教案メモ
- ③ 各回のスライド（タイトル・リンク・種類）
- ④ 参照リンク・メモ

**入れないもの**

- ファイル本体（PDF / PPT / 画像）
- AI（Claude / Gemini）の履歴・提案文
- デモ用サンプルデータ

### 環境

- **開発用 DB と本番用 DB は分ける**
- 接続文字列は `templates/default/.env.local`（ローカル）と Vercel 環境変数（本番）
- **バックアップ**は Neon に任せる。ユーザーデータのダンプを Git に置かない

### 認証・組織

- **Clerk** がユーザー・組織の正本
- DB の `users` / `organizations` / `organization_members` はアプリ用の同期コピー（シラバス紐付け用）

---

## リポジトリ（Git）に入れるもの

- アプリケーションコード
- Drizzle スキーマ・マイグレーション方針
- デモデータ（`demo-syllabus-detail.ts` 等）— **DB には seed しない**
- 図解ページ（`docs/curriculum-tool-overview.html`、`docs/images/`）
- Surge 用コピー（`surge-overview/`）— デプロイ用。編集の正本は `docs/` を推奨

**入れないもの**

- `.env.local`、API キー、DB 接続文字列
- 本番のユーザーデータ・教案のエクスポート
- ビルド成果物（`.next` 等 — `.gitignore` 済み）

---

## ファイル・教材の扱い

### 現在（試作）

- スライド・教材は **Canva / Google Drive 等の URL** を DB に保存
- ファイル本体は外部サービス上に置く

### 将来（合意済み・未実装）

| 項目 | 方針 |
|------|------|
| 方式 | 基本は URL 貼り付け + **たまにアプリ内アップロード**（ハイブリッド） |
| 形式 | **PDF と PPT** |
| 紐づけ | **③ 各回のスライド** のみ |
| 1件あたり | **外部 URL か アップロードのどちらか一方** |
| 保存 | 本体は **Vercel Blob / S3 等**（未定）、DB には **URL のみ** |
| 時期 | **後回し** — 今は URL のみで運用 |

PPT はブラウザ内プレビューが難しい場合があり、ダウンロードリンク中心を想定する。
